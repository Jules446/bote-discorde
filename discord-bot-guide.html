1

Créer une application sur le Developer Portal

▼

Tout commence sur le Discord Developer Portal. C'est ici que vous créez et gérez votre application (le bot).

discord.com/developers/applications
  • Connectez-vous avec votre compte Discord
  • Cliquez sur "New Application" en haut à droite
  • Donnez un nom à votre application (ex : MonBot)
  • Acceptez les CGU et cliquez sur "Create"
  • Vous arrivez sur la page General Information de votre app
💡 Conseil : Ajoutez une icône et une description à votre application — elles apparaîtront sur le profil public de votre bot.
2

Créer le Bot et récupérer le Token

▼

Dans le menu de gauche, cliquez sur "Bot". C'est ici que l'on transforme l'application en bot Discord.

  • Cliquez sur "Add Bot" puis confirmez
  • Donnez un nom au bot (différent de l'application si vous voulez)
  • Cliquez sur "Reset Token" pour générer votre token
  • Copiez immédiatement ce token — il ne sera plus affiché !
⚠️ Ne partagez JAMAIS votre token ! N'importe qui qui le possède peut contrôler votre bot. Si vous le publiez par erreur (GitHub, etc.), réinitialisez-le immédiatement.

Activez aussi les Privileged Gateway Intents selon vos besoins :

PRESENCE INTENT
SERVER MEMBERS INTENT
MESSAGE CONTENT INTENT
GUILD MESSAGES (par défaut)
💡 Pour lire le contenu des messages (et pas seulement les slash commands), activez Message Content Intent.
3

Installer l'environnement de développement

▼

Nous allons utiliser Node.js et la bibliothèque discord.js (la plus populaire).

  • Installez Node.js (version 18 ou supérieure recommandée)
  • Créez un dossier pour votre projet
  • Ouvrez un terminal dans ce dossier et initialisez le projet
  • Installez les dépendances nécessaires
# Initialiser le projet
npm init -y

# Installer discord.js et dotenv
npm install discord.js dotenvbash

Structure recommandée du projet :

📁 mon-bot/
  📁 commands/
    ping.js
    hello.js
  index.js
  deploy-commands.js
  package.json
  .env
  .gitignore

Créez le fichier .env avec vos identifiants :

TOKEN=votre_token_secret_ici
CLIENT_ID=votre_application_id
GUILD_ID=votre_serveur_id_test
⚠️ Ajoutez .env dans votre .gitignore pour ne jamais commiter votre token !

Le CLIENT_ID se trouve dans General Information → Application ID sur le Developer Portal.

4

Coder le bot principal (index.js)

▼

Voici le fichier principal qui démarre le bot et écoute les interactions :

// index.js
require('dotenv').config();
const { Client, GatewayIntentBits, Collection } = require('discord.js');
const fs   = require('fs');
const path = require('path');

// Créer le client Discord
const client = new Client({
  intents: [
    GatewayIntentBits.Guilds,
    GatewayIntentBits.GuildMessages,
    GatewayIntentBits.MessageContent,
  ],
});

// Collection pour stocker les commandes
client.commands = new Collection();

// Charger les commandes depuis le dossier /commands
const commandsPath = path.join(__dirname, 'commands');
const commandFiles = fs.readdirSync(commandsPath)
  .filter(f => f.endsWith('.js'));

for (const file of commandFiles) {
  const command = require(path.join(commandsPath, file));
  if (command.data && command.execute) {
    client.commands.set(command.data.name, command);
  }
}

// Événement : bot connecté
client.once('ready', () => {
  console.log(`✅ Connecté en tant que ${client.user.tag}`);
});

// Événement : slash command reçue
client.on('interactionCreate', async interaction => {
  if (!interaction.isChatInputCommand()) return;

  const command = client.commands.get(interaction.commandName);
  if (!command) return;

  try {
    await command.execute(interaction);
  } catch (err) {
    console.error(err);
    await interaction.reply({
      content: '❌ Erreur lors de l\'exécution.',
      ephemeral: true,
    });
  }
});

// Démarrer le bot
client.login(process.env.TOKEN);js
5

Créer vos premières commandes

▼

Créez un fichier commands/ping.js :

// commands/ping.js
const { SlashCommandBuilder } = require('discord.js');

module.exports = {
  data: new SlashCommandBuilder()
    .setName('ping')
    .setDescription('Répond Pong ! 🏓'),

  async execute(interaction) {
    const latence = interaction.client.ws.ping;
    await interaction.reply(
      `🏓 Pong ! Latence : ${latence}ms`
    );
  },
};js

Créez aussi commands/hello.js avec un embed :

// commands/hello.js
const { SlashCommandBuilder, EmbedBuilder } = require('discord.js');

module.exports = {
  data: new SlashCommandBuilder()
    .setName('hello')
    .setDescription('Dit bonjour avec style !'),

  async execute(interaction) {
    const embed = new EmbedBuilder()
      .setColor(0x5865F2)
      .setTitle('👋 Bonjour !')
      .setDescription(
        `Salut ${interaction.user} ! Je suis opérationnel.`
      )
      .setTimestamp();

    await interaction.reply({ embeds: [embed] });
  },
};js

Aperçu de ce que verra l'utilisateur :

Salut @Utilisateur ! Je suis opérationnel.
6

Déployer les slash commands

▼

Les slash commands doivent être enregistrées auprès de l'API Discord avant d'être utilisables. Créez deploy-commands.js :

// deploy-commands.js
require('dotenv').config();
const { REST, Routes } = require('discord.js');
const fs   = require('fs');
const path = require('path');

const commands = [];
const commandFiles = fs.readdirSync(
  path.join(__dirname, 'commands')
).filter(f => f.endsWith('.js'));

for (const file of commandFiles) {
  const { data } = require(`./commands/${file}`);
  commands.push(data.toJSON());
}

const rest = new REST().setToken(process.env.TOKEN);

(async () => {
  try {
    console.log('⏳ Déploiement des commandes...');
    await rest.put(
      Routes.applicationGuildCommands(
        process.env.CLIENT_ID,
        process.env.GUILD_ID
      ),
      { body: commands }
    );
    console.log('✅ Commandes déployées !');
  } catch (err) {
    console.error(err);
  }
})();js

Exécutez ce script une seule fois (et à chaque modification de commandes) :

node deploy-commands.jsbash
💡 Guild vs Global : Les commandes de guild (serveur) sont instantanées. Les commandes globales peuvent prendre jusqu'à 1 heure pour se propager — utilisez les guild commands en développement.
7

Inviter le bot sur votre serveur

▼

Sur le Developer Portal, allez dans OAuth2 → URL Generator.

  • Sous Scopes, cochez bot et applications.commands
  • Sous Bot Permissions, sélectionnez les permissions nécessaires
  • Copiez l'URL générée en bas de page
  • Ouvrez l'URL dans votre navigateur et choisissez votre serveur

Permissions recommandées pour démarrer :

Send Messages
Embed Links
Read Message History
Manage Messages
Use Slash Commands
Connect (pour les salons vocaux)
💡 L'URL d'invitation ressemble à :
https://discord.com/api/oauth2/authorize?client_id=VOTRE_ID&permissions=XXX&scope=bot+applications.commands

Déployer en production (24h/24)

▼

Pour que votre bot tourne en permanence, vous avez plusieurs options :

Railway / Render / Heroku
Hébergement cloud simple, plan gratuit disponible. Idéal pour débuter.
VPS (DigitalOcean, Contabo, OVH)
Serveur dédié, contrôle total. Utilisez PM2 pour gérer le processus.
Raspberry Pi local
Solution économique à la maison, nécessite une connexion stable.

Avec PM2 sur un VPS :

npm install -g pm2
pm2 start index.js --name "mon-bot"
pm2 save          # Sauvegarder pour redémarrage auto
pm2 startup       # Démarrer au boot du serveurbash

🎉 Votre bot est en ligne !

Publier sur GitHub

▼

Publiez votre projet sur GitHub pour le sauvegarder, le partager ou le déployer facilement.

⚠️ Avant tout : Ne publiez jamais votre fichier .env sur GitHub — il contient votre token Discord secret !

Créez un fichier .gitignore à la racine du projet :

node_modules/
.envbash

Ensuite, créez un nouveau dépôt sur github.com/new (laissez tout par défaut, sans README), puis exécutez ces commandes dans votre terminal :

# 1. Initialiser Git dans votre dossier
git init

# 2. Ajouter tous les fichiers
git add .

# 3. Premier commit
git commit -m "Initial commit"

# 4. Définir la branche principale en "main"
git branch -M main

# 5. Lier votre dépôt GitHub (remplacez l'URL)
git remote add origin https://github.com/VOTRE_PSEUDO/VOTRE_REPO.git

# 6. Publier
git push -u origin mainbash
💡 Connexion GitHub : Lors du git push, GitHub vous demandera un token d'accès (pas votre mot de passe). Créez-en un sur github.com/settings/tokens avec la permission repo.
💡 Mises à jour futures : Pour publier vos modifications après la première fois : 
git add .
git commit -m "Description de la modification"
git push
